User Interaction
User interaction explorer.exe
Direct user action — local or inbound RDP
Shell-launched process. Someone double-clicked, used the Start menu, or is operating via an inbound RDP session. The user is present on this machine as the target/destination.
Interactive session Inbound RDP target Pivot if malware child
RDP client mstsc.exe ★ NEW
This machine initiated an outbound RDP session
The RDP client was used to connect to another host. This machine is the source/pivot point, not the target. Check HKCU\...\Terminal Server Client\Servers for destination history. Workstation origin = strong lateral movement signal.
Outbound RDP Lateral movement source Normal on jump servers
RDP session rdpclip.exe ★ NEW
Active RDP session with clipboard sharing enabled
One instance per live RDP session. Confirms an active or very recent session. Clipboard is a silent exfil channel — no 4663 events for clipboard data movement. Absence means clipboard sharing disabled or non-standard client.
Session presence marker Clipboard exfil channel Masquerading target
Service & Background
Service host svchost.exe
A DLL-based Windows service spawned a child
Check the -k flag to identify the hosting service group. Common for WMI, Task Scheduler, and service-based malware pivots. Always verify the hosting DLL in the registry.
Background service Check hosting DLL WMI/DCOM abuse
SCM services.exe
SCM started an .exe-based service directly
Service Control Manager launched a new service binary. Unexpected children are a strong indicator of malware persistence — register as a service and SCM starts it on boot.
Boot/on-demand service Persistence indicator Verify in registry
Task Scheduler taskeng.exe / taskhostw.exe
A scheduled task fired a child process
Correlate with Event IDs 4698/4702. Children without corresponding tasks in the registry or a matching trigger are a red flag for task-based persistence. Top persistence mechanism in modern intrusions.
Expected on some systems Top persistence vector Check task XML/registry
Script Engines & LOLBins
Shell cmd.exe / powershell.exe
Command interpreter spawned a process
Context is everything. Normal under explorer.exe. Immediate alert under Office apps, browsers, or java.exe. Encoded/obfuscated PowerShell args (-enc) are a classic malware indicator.
Normal under explorer Office/browser parent → alert Check cmdline encoding
Script engine wscript.exe / cscript.exe
VBScript or JScript execution
Rarely legitimate today. Frequently seen in phishing payloads and dropper chains. Especially suspicious as a child of outlook.exe or winword.exe. Often chained with mshta.exe.
Phishing/dropper Living-off-the-land Macro pivot
LOLBin mshta.exe
HTA host — almost always abused
Microsoft HTML Application host. Rarely has legitimate use today. Executes inline VBScript/JScript, often from a remote URL argument. Almost always worth investigating when seen.
High suspicion Remote script execution Inspect URL argument
LOLBin wmic.exe
WMI-based lateral movement or execution
Classic lateral movement via process call create. Spawns processes without touching cmd/powershell. Deprecated in Win11 but still widely present. Check spawning parent and full args.
Lateral movement No cmd/PS artefacts Some admin use
LOLBin certutil.exe ★ NEW
Certificate tool repurposed as downloader or decoder
Legitimate use: manage certificates and CRLs. Abuse: certutil -urlcache -split -f downloads files from the internet; certutil -decode decodes base64-encoded payloads. Both techniques bypass many proxy policies. Any use outside PKI administration is suspicious.
File download Base64 decode Check -urlcache/-decode args
LOLBin rundll32.exe ★ NEW
Arbitrary DLL execution — no standalone PE required
Loads and calls an export from any DLL. Commonly abused to run malicious DLLs without dropping a standalone .exe — evades detections that focus on PE execution. Watch for rundll32 javascript: (scriptlet abuse) or loading DLLs from Temp/AppData.
DLL proxy execution Scriptlet abuse Check DLL path & export
Document exec winword / excel / outlook
Macro or embedded object spawned a process
Office spawning shells, script engines, or network tools is a signature of macro-based malware or document exploits. The chain Office → cmd/powershell/wscript → network tool is one of the most common initial-access patterns.
Macro execution Phishing initial access Check child cmdline
Recon & Enumeration
Recon net.exe / net1.exe ★ NEW
Domain and local account/group enumeration
Built-in tool for managing users, groups, shares, and sessions. net user /domain, net group "Domain Admins", and net localgroup administrators are textbook attacker recon commands. net1.exe is a functional alias sometimes used to evade simple detections on net.exe. Spawning parent is key — recon commands from cmd.exe under an Office app is a red flag.
Domain enumeration Privilege discovery Check full arguments
Defense evasion attrib.exe ★ NEW
File attributes being modified — likely hiding malware artefacts
Sets or clears file attributes: Hidden (+H), System (+S), Read-only (+R). Legitimate admin use exists but is rare in typical operations. Attackers use attrib +H +S to hide dropped files, payloads, or persistence artefacts from casual inspection and Explorer view.
Hiding malware files Check +H +S arguments Rare legitimate use
Persistence Mechanisms
Persistence sc.exe ★ NEW
Service being created, modified, or started via command line
Service Control utility. sc create installs a new service (classic persistence mechanism), sc config modifies an existing one (hijacking), sc start / sc stop controls execution. Watch for services pointing to Temp, AppData, or UNC paths. Correlate with Event ID 7045 (new service installed).
Service-based persistence Service hijacking Correlate 7045
Persistence schtasks.exe ★ NEW
Scheduled task being created or manipulated via command line
The command-line interface to Task Scheduler. /create establishes persistence; /run triggers immediate execution; /delete may indicate cleanup. Frequently used by malware to survive reboots. Correlate with Event ID 4698 (task created) and the XML definition in C:\Windows\System32\Tasks\.
Scheduled persistence Post-exploitation cleanup Correlate 4698/4702
Defense Evasion & Impact
Anti-forensics / Ransomware vssadmin.exe ★ NEW
Shadow copy manipulation — a ransomware and anti-forensics hallmark
vssadmin delete shadows /all /quiet is one of the most recognisable ransomware pre-encryption commands — it destroys all Volume Shadow Copies, eliminating Windows' built-in backup recovery. Also used to resize shadow storage to prevent future snapshots. Any execution outside a documented backup management process is a critical alert.
Ransomware pre-encryption Backup destruction Immediate escalation
VSS service vssvc.exe ★ NEW
Volume Shadow Copy Service provider is active
The legitimate VSS service process — present during snapshot creation or backup operations. Normally benign, but its presence alongside vssadmin.exe or unusual child processes is worth noting. Malware occasionally injects into or masquerades as vssvc.exe to access locked files (e.g., database files, registry hives) during exfiltration. Always verify image path and signature.
Normal during backups Injection target Locked-file access vector
AD / GPO mmc.exe gpmc.msc ★ NEW
Group Policy Management Console opened — domain-wide policy control
Loaded as an MMC snap-in. From here an attacker can create or modify GPOs that execute scripts on every domain-joined machine simultaneously — the highest-impact lateral movement available. Also used to disable AV/EDR, enable WinRM, or add local admin accounts across the estate. Correlate with Event ID 5136 (directory object modified).
Domain-wide execution AV/EDR disable via GPO Correlate 5136
AD / Raw LDAP mmc.exe adsiedit.msc ★ NEW
Raw LDAP attribute editor — DCSync and delegation setup
ADSI Edit gives read/write access to any LDAP attribute directly. Used to grant DCSync rights by modifying nTSecurityDescriptor on the domain object, configure unconstrained/constrained delegation, or set msDS-AllowedToDelegateTo. No PowerShell trace generated. The most dangerous GUI tool in an AD environment.
DCSync rights setup Delegation abuse No PS artefacts
Security & Download
Security check smartscreen.exe ★ NEW
Downloaded file with Mark of the Web was just executed
Fires when a file with Zone.Identifier ADS is run. Confirms external-origin execution. Absence before a suspicious execution means MOTW was stripped — a bypass indicator. The HostUrl field in the ADS contains the original download URL.
Mark of the Web Download exec marker Absence = MOTW bypass
Server & Active Directory
High-value target ServerManager.exe ★ NEW
Server OS + admin account + interactive session — three signals in one
Only exists on Windows Server; auto-launches for admin-level users at logon. On a DC outside business hours this is an immediate escalation. Spawning mmc.exe opens the GUI admin toolchain. On a workstation it should never appear — if it does, it's masquerading malware.
Server OS confirmed Admin-level session Lateral movement destination
AD recon / modification mmc.exe dsa.msc ★ NEW
Active Directory Users & Computers — full domain GUI access
Loads as dsadmin.dll inside mmc.exe. Enables silent DA group adds (4728), password resets (4724), account creation (4720), SPN setting for Kerberoasting, and full domain recon. No PowerShell artefacts. Correlate MMC process timeline against AD audit log to reconstruct actions.
Tier-0 AD access Domain recon & modification GUI — low cmdline footprint
Kernel & Authentication
Auth process lsass.exe
Auth process — should NEVER spawn children
Handles authentication. Never spawns child processes under normal conditions. Any child almost certainly indicates credential dumping (Mimikatz-style) or injection into the process. Also watch for unexpected lsass memory reads (4656/4663 on the process object).
Child = automatic alert Credential dumping Process injection
Logon winlogon.exe
Interactive logon handler
Handles Ctrl+Alt+Delete, logon/logoff, screen lock. Expected children: LogonUI.exe and userinit.exe only. Anything else points to accessibility feature backdoors (Sticky Keys, Narrator) or process injection.
Normal: userinit, LogonUI Other children → alert Accessibility backdoor
Kernel System (PID 4)
Kernel or kernel-mode driver spawned a process
Expected child is smss.exe only. Anything else warrants deep scrutiny — likely a kernel exploit, rootkit, or driver-level persistence. Userland processes with System as parent are a strong indicator of privilege escalation via kernel.
Normal: smss.exe only Other children → rootkit? Kernel exploit indicator
