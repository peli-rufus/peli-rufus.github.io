Active Directory — Attacks, Detection & Forensics Matrix

Attack mechanics, detection event IDs, forensic artefacts, IOCs, and mitigations for common AD attack techniques.

Audit policy prerequisites: Many detection event IDs require non-default audit policy configuration on Domain Controllers. Key policies to enable: Audit Directory Service Access (for EID 4662, 4661), Audit Account Logon (for EID 4768, 4769, 4771, 4776), Audit Logon Events (4624, 4625, 4648), Audit Process Creation with command line logging (4688), and DS Access auditing (5136, 5137, 4929). Without these enabled, the listed event IDs will not be generated. All DC logs should be collected and retained centrally — attackers routinely clear local Security.evtx.

Log locations on DC: C:\Windows\System32\winevt\Logs\Security.evtx (primary), Microsoft-Windows-ActiveDirectory_DomainService%4Diagnostic.evtx (DS diagnostics), System.evtx. NTDS.dit: C:\Windows\NTDS\ntds.dit + SYSTEM hive required for decryption.

Password Spraying credential Attacker tries one or a few common passwords across many accounts — avoids lockout thresholds. Works against NTLM (port 445) and Kerberos (AS-REQ with wrong password). 4625 — failed logon, Sub Status 0xC000006A (wrong password) across multiple accounts
4771 — Kerberos pre-auth failed across multiple accounts from same IP
4776 — NTLM credential validation failure
Spray pattern: many accounts, same timestamp window, same source IP 		Many EID 4625/4771 failures across different usernames from one source IP within minutes. Low failure-per-account count (stays under lockout threshold). Followed by EID 4624 success for one account. Security.evtx (DC) Fine-grained password policies. Azure AD Smart Lockout. Enforce MFA. Monitor for distributed spray (same password, many accounts, varied source IPs). Entra ID Password Protection for on-prem.
GPP Passwords (MS14-025) credential Group Policy Preferences XML files (Groups.xml, ScheduledTasks.xml, Services.xml) stored in SYSVOL can contain AES-256 encrypted passwords — but Microsoft published the static decryption key in 2012. Any authenticated domain user can read SYSVOL and decrypt the cPassword field. 4688 — gpp-decrypt, Get-GPPPassword execution

Credentials in Shares / Object Properties credential Passwords stored in SMB share files (scripts, config files, wiki pages) or in AD object attributes (Description field, info field, comment) in cleartext. Readable by any authenticated user. 5140 / 5145 — file share access and enumeration

4688 — findstr, Select-String, Snaffler.exe execution 		Snaffler.exe or similar share-spider execution. findstr /si password *.txt in Sysmon/4688 process creation. Bulk LDAP reads of user Description fields. Password-like strings in AD Description attributes. Security.evtx
File server access logs		 Audit AD user Description/comment fields for passwords. Restrict authenticated-user read on sensitive shares. Run Snaffler/DeHashed against your own environment proactively.
Pass-the-Hash (PTH) credential Attacker authenticates using a stolen NTLM hash without knowing the plaintext password. Hash extracted from LSASS memory (Mimikatz sekurlsa::logonpasswords) or SAM/NTDS.dit. Used for lateral movement over SMB/WMI/DCOM. 4624 Type 3 — logon with NtLmSsp authentication package (no Kerberos TGT precedes it)
4648 — explicit credential use
4776 — NTLM auth from unexpected host
No preceding 4768 for the same account = PTH indicator 		Type 3 logon + NTLM + no prior 4768 for same account. Multiple lateral Type 3 logons from single source in short window. LSASS access (Sysmon EID 10) prior to network logons. Logon from host that account never previously used. Security.evtx (source & destination DC) Enable Credential Guard. Disable NTLMv1. Enable SMB signing. Tiered admin model — no admin accounts on workstations. Restrict NTLM where Kerberos is available. Monitor Sysmon EID 10 on lsass.exe.
Pass-the-Ticket (PTT) kerberos Attacker injects a stolen Kerberos TGT or service ticket into their own session using Mimikatz kerberos::ptt or Rubeus ptt. Ticket stolen from LSASS memory of another session. Allows access to resources as the ticket owner without their credentials. 4768 — TGT issued on DC
4769 — service ticket requested
4624 Type 9 (NewCredentials) — injected ticket logon
Ticket use from IP/host mismatching the original ticket issuance host 		TGT used from different IP than where it was issued. Service ticket requested without corresponding TGT request. EID 4624 Type 9 logon (createprocesswithlogon / runas). Memory access to lsass.exe (Sysmon EID 10) preceding ticket use. Security.evtx (DC) Credential Guard prevents LSASS ticket theft. Short Kerberos ticket lifetime (default 10 hours — do not extend). Monitor Sysmon EID 10 on lsass.exe. Restrict who can read tickets from LSASS.
Overpass-the-Hash (OPtH) kerberos Uses a stolen NTLM hash to request a Kerberos TGT — converts the hash into a fully functional Kerberos ticket. Executed via Mimikatz sekurlsa::pth or Rubeus asktgt /rc4. Produces a TGT that can be used like a normal Kerberos session — avoids NTLM detection. 4768 — TGT request using RC4 encryption (etype 23 = 0x17) instead of AES — anomalous if AES should be used
4624 Type 9 — new credentials logon type
4688 — Mimikatz / Rubeus process creation 		AS-REQ using RC4 (etype 0x17) for account that normally uses AES. EID 4768 from host that account does not normally log into. Type 9 logon immediately followed by Kerberos-authenticated activity (bypassing normal Type 2 interactive logon). Security.evtx (DC) Enforce AES-only Kerberos (disable RC4/DES). Credential Guard. Monitor for RC4 TGT requests from accounts that have AES keys available. Alert on Type 9 logons from privileged accounts.
Kerberos Attacks
TechniqueHow it worksKey detection event IDsIOC patternsLog sourceMitigation
Kerberoasting kerberos Any authenticated domain user can request a Kerberos service ticket (TGS) for any SPN. The ticket is encrypted with the service account's NTLM hash. Attacker requests ticket offline, extracts hash, and cracks it. No elevated privileges required. 4769 — service ticket requested with RC4 encryption type (0x17) — AES-capable accounts should never use RC4
Burst of 4769 RC4 requests across multiple service accounts from single source
4688 — Rubeus.exe, Invoke-Kerberoast, GetUserSPNs.py 		Multiple EID 4769 RC4 requests in short window from one host. Service ticket requests for rarely-used service accounts. Ticket Options value 0x40810000 (common Kerberoasting tool flag). Rubeus/Invoke-Kerberoast in process/PS logs. Security.evtx (DC) Use gMSA — auto-rotating 240-char passwords, not crackable. Enforce AES-only (disable RC4) — cracks become infeasible. Audit SPNs on user accounts. Monitor 4769 RC4 requests. Set canary SPN account that should never be requested.
AS-REProasting kerberos Accounts with DONT_REQ_PREAUTH (UF 0x400000) set do not require Kerberos pre-authentication. Attacker sends AS-REQ for these accounts without any credentials — DC returns AS-REP encrypted with the account's hash. Attacker cracks offline. Requires zero authentication. 4768 — AS-REQ with no pre-authentication (PreAuthType = 0) for accounts that have pre-auth disabled
4625 — failed pre-auth if account does require it (failed roasting attempt)
4688 — Rubeus.exe asreproast, GetNPUsers.py 		EID 4768 with Pre-Authentication Type 0 and RC4 encryption from unexpected source. Multiple AS-REQs for different accounts from same IP without corresponding logon activity. DONT_REQ_PREAUTH set on any account (should be rare/never). Security.evtx (DC) Audit and disable DONT_REQ_PREAUTH on all accounts — it should never be set without documented business need. Strong passwords on affected accounts. Monitor EID 4768 PreAuthType=0.
Golden Ticket kerberos Attacker forges a TGT using the krbtgt account's NTLM hash (obtained via DCSync or NTDS.dit dump). Forged TGT is cryptographically valid from the DC's perspective — bypasses all normal authentication checks. Can impersonate any user with any group membership. Survives password resets (krbtgt must be reset TWICE). 4769 — service ticket requested using a TGT that has no corresponding 4768 on the DC
4672 — special privileges assigned at logon with forged ticket
Ticket lifetime anomaly: default 10hrs — forged tickets often set to 10 years
4624 — logon event where ticket domain differs from account domain 		4769 with no preceding 4768 for same account. Ticket lifetime >10 hours (forged tickets default to 10 years in Mimikatz). Account claiming membership in groups it does not belong to (e.g. RID 519 Enterprise Admins for non-EA account). Domain field in 4624 is FQDN rather than NETBIOS (Mimikatz default). Security.evtx (DC) Rotate krbtgt password TWICE (10+ hours apart) to invalidate all existing TGTs. Enable Protected Users group for privileged accounts. Monitor 4769 without preceding 4768. Defender for Identity Golden Ticket detection. krbtgt rotation after any suspected DC compromise.
Silver Ticket kerberos Attacker forges a service ticket (TGS) using the target service account's NTLM hash — without touching the DC or KDC. Grants access directly to a specific service (e.g. CIFS, HOST, MSSQLSvc). Leaves NO KDC log entries — the DC is never contacted. No DC event logs generated — Silver Ticket bypasses the KDC entirely
4624 on the target service host — logon with Kerberos but no DC authentication record
4634 — logoff shortly after with no corresponding DC activity
Network: Kerberos AP-REQ directly to service with no AS-REQ/TGS-REQ to DC 		Service authentication with no corresponding DC ticket issuance. Account using Kerberos to access service it has never accessed before. PAC validation disabled (Silver Tickets often have invalid PAC). Network sensor: Kerberos AP-REQ without preceding TGS exchange to DC. Security.evtx (target host)
Network Kerberos traffic		 Enable PAC validation (KDC PAC validation — requires registry key on services). Strong, rotating service account passwords. Credential Guard. Defender for Identity Silver Ticket detection via PAC inspection.
Kerberos Constrained Delegation (KCD) abuse kerberos Accounts/computers with constrained delegation (msDS-AllowedToDelegateTo) can request service tickets on behalf of any user to specific services. If the account is compromised, attacker can impersonate any domain user (including DA) to the allowed services using S4U2Proxy. Protocol transition (S4U2Self) allows impersonation without requiring the victim's ticket. 4769 — S4U2Proxy service ticket requests where Transited-Services field is populated
4769 — S4U2Self ticket request (service requesting ticket for another user to itself)
4688 — Rubeus.exe s4u, getST.py 		4769 with Transited-Services field containing unexpected accounts. Service ticket for high-privilege user (DA, EA) requested via S4U from a service account. Delegation to sensitive services (CIFS on DC, LDAP on DC). Rubeus s4u syntax in process logs. Security.evtx (DC) Mark sensitive accounts as "Account is sensitive and cannot be delegated." Add DAs to Protected Users group. Audit msDS-AllowedToDelegateTo attributes. Prefer Resource-Based Constrained Delegation (RBCD) over classic KCD. Never configure delegation to DC services.

TechniqueHow it worksKey detection event IDsIOC patternsLog sourceMitigation
Unconstrained Delegation kerberos Computers with TRUSTED_FOR_DELEGATION (UF 0x80000) store TGTs of all users who authenticate to them in LSASS memory. An attacker who compromises such a host can extract all cached TGTs. Combined with coercion (PrinterBug, PetitPotam), attacker can coerce DC machine account to authenticate — capturing its TGT and achieving DCSync-equivalent access. 4769 — forwardable TGT requested for user authenticating to delegation host
4624 Type 3 on delegation host from coerced DC machine account
4688 — Rubeus.exe monitor /targetuser: on delegation host
Sysmon EID 10 — lsass.exe access on delegation host 		DC machine account (DCNAME$) authenticating to a non-DC host (coercion result). Rubeus monitor or dump on unconstrained delegation server. TGT for high-privilege account extracted from non-DC LSASS. Delegation host suddenly generating large volume of Kerberos requests. Security.evtx (DC & delegation host) Eliminate unconstrained delegation — replace with constrained or RBCD. Audit: Get-ADComputer -Filter {TrustedForDelegation -eq $true}. Mark sensitive accounts as "cannot be delegated." Add DAs to Protected Users (blocks delegation). Disable Print Spooler on DCs to prevent coercion.

Net logon log C:\Windows\debug\netlogon.log (DC) Detailed authentication debugging log — NTLM and Kerberos failures with source IP. Not enabled by default but invaluable when present. Records failed authentications that may not appear in Security.evtx. Enable: nltest /dbflag:0x2080ffff. Disable after collection to avoid disk fill. Useful for correlating spray/relay source IPs when Security log is incomplete.
