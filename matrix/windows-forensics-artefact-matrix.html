Evidence of Execution
Artefact Proves execution? Proves presence? Survives deletion? Timestamps Run count? Caveats
Prefetch (.pf) execution Strong Yes Yes Last 8 run times Yes Disabled on Server OS by default. Only last 8 timestamps stored. Enable via registry if needed.
UserAssist execution Strong (GUI only) Yes No Last run time Yes CLI executions not recorded here. Values are ROT13 encoded — Registry Explorer decodes automatically.
ShimCache presence Not alone Strong Yes Last modified time No Presence ≠ execution. Executed flag is NA on Win10+. Must correlate with Prefetch or BAM to claim execution.
AmCache.hve execution Strong Yes Yes First run + install time No SHA1 hash available for IOC matching. Schema differs between Win8.1 (Root\File) and Win10/11 (Root\InventoryApplicationFile).
BAM / DAM execution Strong Yes Partial Last execution time No Per-SID entries — good for attribution. Resets on some Windows updates. Covers background and foreground processes.
Windows 10 Timeline execution Yes Yes Partial Focus time No SQLite ActivitiesCache.db. Removed/limited in Win11. May sync across devices if MS account is linked.
PcaAppLaunchDic.txt execution Yes Yes Partial Yes — per entry No Windows 11 22H2+ only. Pipe-delimited path|timestamp format. Not yet parsed by most forensic tools.
IconCacheDB presence Not alone Yes No No No Written at shutdown/reboot only — not real-time. Proves binary was present after Explorer browsing. Correlate with Prefetch + AmCache.
MUICache execution Partial — Explorer-launched only Yes No No timestamps No Explorer-launched binaries only — does not capture cmd/PS/service execution. No per-entry timestamps. Entry survives binary deletion. Blank or @ PE description = no resource section (suspicious). Vista+: HKCU\Software\Classes\Local Settings\Software\Microsoft\Windows\Shell\MuiCache in UsrClass.dat. XP: HKCU\Software\Microsoft\Windows\ShellNoRoam\MUICache in NTUSER.DAT. Tools: Registry Explorer, RegRipper (rip.pl -r UsrClass.dat -p muicache), KAPE Target: RegistryHives.
RecentApps (AppLaunched / AppSwitched) execution Yes — AppLaunched proves execution Yes Yes — LastAccessTime per app Yes — LaunchCount per app No Windows 10+ only — not on Server OS or Win7/8. Stored in NTUSER.DAT. AppLaunched = executed via shell, survives deletion. AppSwitched = user actively interacted (Alt-Tab/taskbar) — AppLaunched with no AppSwitched = automated/background execution (service, scheduled task, C2). Path: HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\RecentApps. Tools: Registry Explorer, RegRipper (rip.pl -r NTUSER.DAT -p recentapps), KAPE Target: RegistryHives.
File & Folder Activity
Artefact Proves file access? Survives deletion? Timestamps Full path? Caveats
LNK files presence Strong Yes Created + accessed Yes Contains volume serial — links file to specific physical drive. Auto-created by Explorer only, not CLI access.
Jump Lists presence Strong Yes Yes Yes Per-application via AppID. AutoDest + CustomDest. Records network share paths — useful for lateral movement cases.
RecentDocs (registry) presence Yes Partial Last opened Filename only Extension-specific subkeys. MRU ordered. Does not store full path — correlate with LNK files for full path.
ShellBags presence Folder browsing Yes First + last interacted Yes Survives folder deletion. Records network share and external drive access. Primary source is USRCLASS.DAT.
Office MRU presence Yes Partial Yes Full path Version-specific registry path. O365 uses LiveID subkey path. Separate from Windows RecentDocs.
WordWheelQuery presence Search terms only Partial Key LastWrite only No Binary encoded. Reveals what the user was searching for — high value in insider threat and data theft cases.
IE/Edge WebCache presence Local files too Partial Yes Yes (file:/// prefix) Records locally opened files, not just web history. file:/// entries indicate local file access. Parse with Autopsy.
ComDlg32 (CIDSizeMRU · LastVisitedPidlMRU · OpenSavePidlMRU) presence Open/save dialog activity Partial Key LastWrite · MRU order Full path (OpenSave) CIDSizeMRU = recently launched apps globally. LastVisitedPidlMRU = executable + last directory used per app. OpenSavePidlMRU = full paths of files opened/saved per extension. Does not prove execution — proves file interaction via dialog box. Registry path: NTUSER.DAT\Software\Microsoft\Windows\CurrentVersion\Explorer\ComDlg32.
Recycle Bin ($I / $R) presence Deletion evidence Until emptied Deletion timestamp Original full path SID subfolder identifies which user deleted the file. $I = metadata, $R = file content. Tool: RBCmd.exe.
Windows.edb (Search Index) presence File existence Yes Indexed timestamp Yes Proves a file existed and was indexed even if subsequently deleted. ESE database. Tools: esedatabaseview, sidr.exe.
NTFS Artefacts
Artefact Proves presence? Survives deletion? Timestamps Detects tampering? Caveats
$MFT presence Strong Until overwritten MACB — two full sets SI vs FN comparison Every file/directory has an MFT entry. Deleted entries persist until the slot is reused. Tool: MFTECmd.exe.
$STANDARD_INFORMATION timestamps (SI) Yes In MFT MACB (what Explorer shows) Easily stomped Can be modified by any user with timestomping tools. Do not rely on SI timestamps alone in adversarial cases.
$FILE_NAME timestamps (FN) Yes In MFT MACB (in MFT only) Kernel access required to modify SI vs FN mismatch = strong timestomping indicator. Not visible in Explorer — requires MFTECmd.exe to extract.
$UsnJrnl ($J stream) presence Yes Wraps ~72 hrs Per-operation timestamp Rename detection Records every create/rename/delete/modify operation. Best source for rename and deletion evidence. Tool: MFTECmd.exe -f $J.
$LogFile presence Recent ops Very short window Transaction log entries Partial NTFS transaction log — very short retention. Useful for immediate post-incident recovery of recent file system changes.
Alternate Data Streams (ADS) presence Hidden data Partial Yes Partial Zone.Identifier stream proves internet origin (Mark of the Web). Attackers hide payloads in ADS. Tools: streams.exe, MFTECmd.
$I30 (Directory Index) presence Deleted filenames Until index page reuse FN timestamps per entry Yes — with FN timestamps B-tree index of directory entries. Slack space in unallocated index pages retains deleted file names, sizes, and $FILE_NAME timestamps even after MFT record reallocation and $UsnJrnl rollover. Key uses: (1) recover names of deleted attacker tools from staging directories, (2) detect timestomping — $I30 FN timestamps are harder to modify than SI timestamps; SI/FN mismatch = stomping. Each entry contains all four FN timestamps (crtime, mtime, atime, ctime) + filename + file size + MFT reference. Path: $I30 named stream on any directory MFT record ($INDEX_ROOT 0x90 + $INDEX_ALLOCATION 0xA0). Tools: MFTECmd (MFTECmd.exe -f "$MFT" --csv . --csvf mft.csv), INDXParse.py, icat (TSK: icat image.dd <inode>-144-4 > i30.bin), KAPE Target: MFT. Vol3: windows.mftscan.MFTScan.
USB & External Devices
Artefact Proves connection? Survives removal? Timestamps Identifies device? Caveats
USBSTOR (registry) Strong Yes First connect + last connect + last removal Vendor / product / serial Timestamps via {83da6326} property key subvalues 0064/0066/0067. Mass storage class devices only.
USB (registry) Yes Yes Plug-in time VID / PID All USB device classes — keyboards, cameras, dongles, not just storage. Broader scope than USBSTOR.
MountedDevices (registry) Drive letter mapping Yes No Volume signature Links drive letters (e.g. E:) to device signatures. Essential for reconstructing exfiltration path and correlating LNK volume serials.
MountPoints2 (NTUSER.DAT) User attribution Yes No Yes Per-user registry key — proves which user account interacted with the device. Critical for attribution in insider threat cases.
Setupapi.dev.log First connection Yes First seen timestamp Device ID + serial Text log at C:\Windows\inf\. Shows device ID and serial number on first attachment. Not updated on subsequent connections.
LNK / shortcut files Indirect Yes File access time Volume serial number Contains volume serial of the source drive. Cross-reference with MountedDevices to confirm which physical device a file came from.
Network & Browser
Artefact Proves network activity? Retention Timestamps Key forensic value Caveats
SRUM (SRUDB.dat) network Strong 30–60 days Daily buckets Bytes sent/received per process per day. Quantifies data volume for exfiltration investigations. Locked on live system — collect with KAPE. ESE database requiring esentutl recovery if dirty.
NetworkList (registry) network Yes Persistent First + last connected Every network ever connected — SSID, GUID, managed vs unmanaged (domain vs public). Key LastWrite = last connection time. Managed = domain-joined network. Unmanaged = public/home.
tsclient MRU (registry) network RDP history Persistent Key LastWrite RDP target hostnames/IPs. UsernameHint for account attribution. CertHash for endpoint verification. Client-side only. Combine with Event ID 1149 (TerminalServices log) for server-side confirmation.
RDP Bitmap Cache network Session content Persistent No Partial screenshots of remote desktop sessions — can reveal attacker activity on the remote host. Binary tile format. Reconstruction is partial. Location: %LOCALAPPDATA%\Microsoft\Terminal Server Client\Cache\. Tool: bmc-tools.
PuTTY SshHostKeys network SSH history Persistent Key LastWrite Fingerprints of SSH servers connected to. FileZilla shares this same cache. NTUSER.DAT hive — per-user. Software\SimonTatham\PuTTY\SshHostKeys.
Browser History network Web activity Until cleared Visit timestamps URLs, visit count, typed vs clicked. Identify phishing URLs, C2 panels, staged payloads. Firefox: places.sqlite. Chrome/Edge: History (SQLite). Timestamps in WebKit format for Chrome. Tool: Hindsight.
Browser Downloads network File origin Until cleared Download time Source URL + local save path. Correlate with Prefetch for evidence of execution post-download. Chrome/Edge: downloads table in History SQLite. Firefox: moz_annos table in places.sqlite.
CryptnetUrlCache network Crypto/cert URLs Persistent Yes URLs contacted for cert validation or payload staging. certutil abuse leaves entries here. Per-user profile at %APPDATA%\LocalLow\Microsoft\CryptnetUrlCache\. certutil in execution artefacts + entries here = high-confidence IOC.
Windows Firewall Log network Connection attempts Limited (log rotation) Per-packet timestamp ALLOW/DROP per connection with source/dest IP and port. Inbound lateral movement detection. Logging disabled by default in some configs. Location: System32\LogFiles\Firewall\pfirewall.log.
SUM Database network Client access 2-year rolling window InsertDate + LastAccess Which clients/accounts connected to which server services — SMB, RDP, IIS, AD DS. Maps lateral movement. Windows Server 2012+ only. ESE database at System32\LogFiles\SUM\. Requires esentutl recovery if dirty. Tool: SumECmd.
Persistence Mechanisms
Artefact Admin required? Survives reboot? Timestamps Key forensic value Caveats
Run / RunOnce keys persistence HKLM = yes · HKCU = no Yes Key LastWrite Simplest persistence mechanism. HKCU Run writable without admin rights — very common malware location. RunOnce deletes the entry after first execution — may be gone by collection time. Check RegBack hive backups.
Services (registry) persistence Yes Yes Key LastWrite ImagePath = executable. ServiceDll = actual DLL for svchost-hosted services. Start value reveals boot behaviour. For shared-process services: check Parameters\ServiceDll — ImagePath just shows svchost.exe. Start 0x00/0x01 = kernel driver level.
Scheduled Tasks (XML + registry) persistence Depends on principal Yes Creation + last run XML survives registry cleaning. Registry TaskCache survives XML deletion. Always check both locations. Event IDs 106/140/141/200/201 (TaskScheduler log) and 4698/4699/4702 (Security log) for creation/deletion timeline.
Winlogon hijacking persistence HKLM = yes · HKCU = no Yes Key LastWrite Userinit and Shell values run for every user at logon. HKCU version requires no admin rights. Default Userinit = userinit.exe, (trailing comma is normal). Any addition after the comma is malicious.
IFEO (Debugger hijacking) persistence Yes Yes Key LastWrite Replaces any named process with attacker binary. Accessibility binaries (sethc.exe, osk.exe) enable logon-screen backdoor. Image File Execution Options. Common targets: sethc.exe, osk.exe, magnify.exe, utilman.exe — all launchable from locked screen.
Startup folders persistence AllUsers = yes · per-user = no Yes File creation time Simplest technique — any file type dropped here executes at logon. No registry knowledge required. Per-user: AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup. AllUsers: ProgramData\Microsoft\Windows\Start Menu\Programs\StartUp.
WMI Subscriptions persistence Yes Yes No timestamps Fileless persistence — survives registry and file system cleaning. Three components: EventFilter + EventConsumer + FilterToConsumerBinding. Stored in WMI repository (System32\wbem\Repository\), not registry. Tool: PyWMIPersistenceFinder, Velociraptor VQL artifact Windows.Persistence.PermanentWMIEvents.
AppInit_DLLs persistence Yes Yes Key LastWrite DLLs loaded into every process that loads user32.dll — very broad injection surface. Deprecated/blocked with Secure Boot on Win8+. Still present on legacy or misconfigured systems. High signal if populated.

Computer Name SYSTEM CurrentControlSet\Control\ComputerName\ComputerName Confirms you are analysing the correct machine. First step in any investigation.
Time Zone SYSTEM CurrentControlSet\Control\TimeZoneInformation Essential for timeline normalisation — some artefacts use local time, others UTC. Must establish before building any timeline.

Network Interfaces SYSTEM CurrentControlSet\Services\Tcpip\Parameters\Interfaces IP config, DHCP, DNS per interface. Confirms machine identity and network context.

LanmanServer Shares SYSTEM CurrentControlSet\Services\LanmanServer\Shares Active network share names hosted by this machine. Useful for understanding what data was exposed and accessible over the network.
MountPoints2 NTUSER.DAT Software\Microsoft\Windows\CurrentVersion\Explorer\MountPoints2 Per-user evidence of USB device and network share access. Attribution artefact — proves which user account interacted with an external resource.
Firewall Rules SYSTEM ControlSet001\Services\SharedAccess\Parameters\FirewallPolicy\FirewallRules Active firewall rules. Attacker-added allow rules for C2 ports are a high-fidelity indicator of compromise.
Email Artefacts
Artefact Proves email activity? Survives deletion? Timestamps Key forensic value Caveats

Thunderbird MBOX Yes Until wiped Yes Plaintext format — grep-friendly for large-scale keyword or IOC searching without tools. Use mboxgrep for CLI searching or import into Thunderbird. Headers and body fully readable as plain text.
Memory Artefacts (Disk-Resident)
Artefact Contains memory content? Survives reboot? Analysable with Volatility? Key forensic value Caveats
hiberfil.sys Full RAM snapshot Yes — until next hibernate Yes — after conversion Complete RAM image from last hibernation. Contains processes, network connections, credentials, encryption keys, and decrypted strings from malware. Best disk-resident memory source. Compressed — must convert with Volatility's imagecopy or hibernate2dmp before analysis. Location: C:\hiberfil.sys. Only present if hibernation is enabled.
pagefile.sys Memory fragments only Yes — until overwritten No — not full pages Memory fragments paged to disk when RAM was full. Can contain passwords, URL strings, document content, and attacker command output. Best analysed with strings search. Non-sequential 4KB chunks — cannot reconstruct process structures. Location: C:\pagefile.sys. Tool: bulk_extractor, strings, FTK Imager for extraction.
swapfile.sys UWP app memory only Yes No Stores suspended Microsoft Store (UWP) app state. Can reveal activity of Store apps that are no longer running. Windows 10+ only. Limited to Store apps. Location: C:\swapfile.sys alongside pagefile.sys.
Crash dumps (minidump / kernel) Partial or full Yes Yes BSOD crash dumps can reveal malware that triggered a kernel exception or anti-analysis technique. Faulting module named in dump header. Minidumps: C:\Windows\Minidump\. Full kernel dump: C:\Windows\MEMORY.DMP. Not always present. Requires crash to have occurred.
PowerShell Artefacts
Artefact Proves execution? Survives deletion? Has timestamps? Key forensic value Caveats
ConsoleHost_history.txt Yes — commands typed No — easily deleted No per-command timestamps Plaintext log of every command typed interactively. Attacker tooling, download cradles, lateral movement commands. Per-user profile. Location: %APPDATA%\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt. File $SI modified time = last command run. Can be disabled with Set-PSReadlineOption -HistorySaveStyle SaveNothing — absence of file is itself suspicious.
PowerShell Script Block Logging (Event ID 4104) Strong Until log rotation Yes — per event Logs decoded script block content including AMSI-bypassed and obfuscated scripts. Captures what actually executed, not just what was typed. Highest fidelity PowerShell artefact. Requires Group Policy enablement. Log: Microsoft-Windows-PowerShell/Operational. Event ID 4104 = script block. 4103 = module logging. Not enabled by default on all systems.
PowerShell Transcripts Strong Until deleted Session start/end timestamps Full input and output of a PowerShell session including command results — not just commands. Closest to a session recording. Requires GPO enablement (Transcription policy). Location is administrator-configured — typically C:\Users\<user>\Documents\PowerShell_Transcripts\ or central share. Not on by default.
PowerShell Operational Log (Event ID 400/600) Yes Until log rotation Yes Engine start (400) and stop (600). HostApplication field reveals what process launched PowerShell — distinguishes interactive session from malware invoking powershell.exe. Log: Windows PowerShell (classic). Event ID 400 records the PowerShell version, which can detect downgrade attacks (attacker forcing v2 to bypass logging).

Lateral Movement Artefacts
Technique Host artefacts (source) Host artefacts (destination) Event log evidence Caveats
PsExec ShimCache / Prefetch for psexec.exe. tsclient MRU if via RDP first. NTUSER Run key if persistent. PSEXESVC.exe in System32. Service creation in SYSTEM hive. Prefetch for PSEXESVC.exe. Source: 4648 (explicit creds). Dest: 7045 (service install), 5140 (ADMIN$ share access), 4624 Type 3, 4688 (PSEXESVC process). Named pipe artefact: \\target\pipe\PSEXESVC. PSEXESVC.exe deleted after session but Prefetch entry persists.
RDP (Remote Desktop) tsclient MRU (target hostname). RDP Bitmap Cache (session screenshots). ShimCache for mstsc.exe. Event ID 1149 (connection). 4624 Type 10 (logon). RDP session registry keys. Prefetch for rdpclip.exe, tsclient. Source: 4648. Dest: 1149 (TerminalServices), 4624 Type 10, 4778/4779 (reconnect/disconnect), 4647 (logoff). Bitmap cache reconstructs partial session screen. HKCU\Software\Microsoft\Terminal Server Client\Default on source.
WMI (wmic / Invoke-WMIMethod) Prefetch for wmic.exe. PowerShell history / Script Block log if PS-based. ShimCache. WMI repository activity. Process creation as child of WmiPrvSE.exe (4688). No service creation — harder to detect. Source: 4648. Dest: 4624 Type 3, 4688 (child of WmiPrvSE.exe), Event ID 5857/5858/5859/5860/5861 in WMI-Activity/Operational log. Fileless — no binary dropped on destination. WMI-Activity log Event ID 5861 = new permanent subscription created (persistence).
Pass-the-Hash (PTH) Mimikatz Prefetch / ShimCache. lsass.exe access in Sysmon Event 10. 4624 Type 3 with NtLmSsp authentication package. No Kerberos TGT (4768) preceding it. 4624 Type 3 + NTLM + no prior 4768 = PTH indicator. 4648 on source. Look for lateral movement pattern: multiple Type 3 logons from same source in short window. Legitimate NTLM Type 3 logons exist — context and clustering matter. Workgroup environments use NTLM legitimately.
PowerShell Remoting (WinRM) PowerShell history / Event ID 4103/4104 on source. Prefetch for wsmprovhost.exe. wsmprovhost.exe process creation (4688). 4624 Type 3. WinRM service activity. Event ID 6 in PowerShell/Operational. Source: 4648, 4103/4104. Dest: 4624 Type 3, 4688 (wsmprovhost.exe as child of svchost.exe), Microsoft-Windows-WinRM/Operational log. Requires WinRM enabled on destination. Default port 5985 (HTTP) / 5986 (HTTPS). Encrypted — network capture shows no plaintext.

Credential Artefacts
Artefact Contains credentials? Survives reboot? Admin required to access? Key forensic value Caveats
Windows Credential Manager Yes — encrypted Yes No (user-accessible) Saved passwords for websites, network shares, RDP targets, and applications. Reveals what resources the user authenticated to and saved credentials for. Location: %APPDATA%\Microsoft\Credentials\ and %LOCALAPPDATA%\Microsoft\Credentials\. Encrypted with DPAPI. Tool: Mimikatz dpapi::cred or offline with master key. Registry: HKCU\Software\Microsoft\Credentials.

Possession of master key + SYSTEM/SECURITY hive enables offline credential decryption. Location: %APPDATA%\Microsoft\Protect\<SID>\. Encrypted with user password hash derivation. Requires SYSTEM + SECURITY hives for offline decryption. Tools: Mimikatz, dpapick.
SAM + SYSTEM hive (NTLM hashes) NTLM password hashes Yes Yes — SYSTEM privilege Local account password hashes for offline cracking or Pass-the-Hash. Combined with SYSTEM hive bootkey for decryption. Locked on live system — requires VSS or raw disk access. Offline: both SAM and SYSTEM hives required. Tools: secretsdump (Impacket), samdump2. Protected by Credential Guard on modern systems.
Browser saved passwords (Login Data) Yes — DPAPI encrypted Yes No Saved website credentials. Reveals phishing site credential submission if user saved creds to attacker-controlled site. Chrome/Edge: %LOCALAPPDATA%\Google\Chrome\User Data\Default\Login Data (SQLite). Passwords encrypted with DPAPI or App-Bound Encryption (Chrome v127+). Live system: tools like SharpChrome. Dead box: DPAPI key chain required.
LSA Secrets (SECURITY hive) Service account creds Yes Yes — SYSTEM Cached domain credentials, service account passwords, DPAPI system key, VPN/dial-up credentials. High-value target for attackers. Location: HKLM\SECURITY\Policy\Secrets\. Locked on live system. Offline: SYSTEM + SECURITY hives. Tools: secretsdump, Mimikatz lsadump::secrets.

httperr.log (HTTP.sys error log) web Yes — requests rejected before reaching IIS Yes — written by kernel driver UTC per entry Written by HTTP.sys (kernel-mode driver) — not IIS. Captures requests that never reached the IIS worker process. Unique forensic value: records exploit attempts invisible to IIS W3C logs, and RCE attempts that crash the app pool (crash appears here, IIS log has no entry because w3wp.exe died before writing). Fields: c-ip, cs-method, cs-uri, s-reason (key field), s-queuename.

s-reason forensic values:
• Timer_ConnectionIdle — scanner probe / keep-alive timeout
• Verb — unusual HTTP method (PROPFIND/SEARCH/TRACK = WebDAV CVE, ProxyShell)
• URL — malformed URL, path traversal, %2e/%00 encoded payload rejected by HTTP.sys
• Header — invalid headers, payload-in-header attacks (Log4Shell in User-Agent or X-Forwarded-For)
• Connection_Abandoned_By_AppPool — w3wp.exe crashed mid-request (RCE trigger — correlate with 4688 child processes)
• AppOffline — app pool offline when request arrived (post-crash state)		 Path: C:\Windows\System32\LogFiles\HTTPERR\httperr<N>.log (rotated by size ~1MB)
Applies to all HTTP.sys consumers: IIS, Exchange, SharePoint, WinRM (5985), RDP over HTTP (3389)

Collection: KAPE (Target: IISLogFiles) or manual copy

Tools & commands:
• PowerShell parse (handles streamid column correctly):

• Log Parser 2.2 (pre-HTTP API 2.0 only — streamid breaks native HTTPErr mode): use TSV mode with custom header file

Caveats: Timestamps UTC. streamid column (HTTP API 2.0, Win2012+) breaks Log Parser's native -i:HTTPErr — use PowerShell or TSV workaround. Correlate same c-ip across httperr (probe errors) and IIS W3C (subsequent successful requests) to identify probe-then-exploit pattern.

QMGR database (qmgr.db / qmgr0.dat) primary Yes — job definition, source URL, destination path, owner SID Yes — persists across reboots CreationTime, ModifiedTime per job The Queue Manager database is the single most important BITS artefact. Contains active and recoverable deleted BITS job records including: JobType (download/upload), JobName, source URL, local destination path, owning user SID, creation and modification timestamps, job state, and crucially SetNotifyCmdLine — the command executed when a job completes or errors. Anti-Forensics Indicators
Technique What the attacker does How to detect it Residual evidence
Timestomping Modifies $STANDARD_INFORMATION (SI) timestamps to blend in with legitimate files. Compare SI timestamps vs $FILE_NAME (FN) timestamps in MFT. SI/FN mismatch = strong indicator. Tool: MFTECmd.exe outputs both sets. FN timestamps cannot be modified without kernel-level access. SI creation time earlier than FN creation time is impossible legitimately — definitive indicator.
Log clearing Clears Security, System, or Application event logs to erase activity trail. Event ID 1102 (Security log cleared) persists in the newly cleared log. Event ID 104 for other logs. Unexplained gap in event log timeline is itself suspicious. Log clearing is all-or-nothing — cannot selectively delete events via standard Windows. SRUM, Prefetch, and registry artefacts survive log clearing.
VSS deletion Deletes Volume Shadow Copies to prevent forensic recovery of previous file/registry states. Event ID 8222 (VSS) for snapshot deletion. vssadmin.exe or wmic shadowcopy delete in Prefetch, PowerShell history, 4688 process creation logs. Common ransomware pre-encryption step. Absence of VSS snapshots on a system that should have them is suspicious. SRUM may show vssadmin execution.
Prefetch disabling Sets HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management\PrefetchParameters\EnablePrefetcher to 0. Registry key value of 0 = disabled. Key LastWrite timestamp shows when it was changed. Absence of Prefetch files on a workstation OS is suspicious. Prefetch is enabled by default on workstations. Disabled by default on Server OS — absence on a server is normal.
PowerShell history deletion Deletes or renames ConsoleHost_history.txt, or sets SaveNothing history policy. Absence of history file where PS was clearly used is suspicious. UsnJrnl may show deletion of the file. Non-PowerShell process (Explorer.exe, cmd.exe) modifying the history file = strong IOC. Script Block Logging (Event ID 4104) survives history file deletion — captures commands at execution time, not after. Enable SBL via GPO to counter this technique.
Tool execution from memory / fileless Executes payload entirely in memory — never writes binary to disk. PowerShell, WMI, .NET reflection. Script Block Logging (4104) captures decoded payload. AMSI telemetry. Memory forensics (Volatility malfind). No Prefetch or AmCache entry for the payload. SRUM still records network activity by process even for fileless execution. Named pipes in SRUM network data can indicate C2 beaconing process.
Alternate Data Streams (ADS) for hiding Hides malicious payload in NTFS ADS of a legitimate file (e.g. readme.txt:payload.exe). MFTECmd shows ADS streams. streams.exe (Sysinternals) enumerates ADS on live system. ADS larger than parent file is a red flag. Zone.Identifier ADS is legitimate (Mark of the Web from internet downloads). Other ADS names on executables or documents are suspicious.
